# HTTP requests

### APIs

Retrieve JSON data for analysis by making HTTP requests to different APIs (clouds). Explore and:

- evaluate prominent APIs from Google or ChatGPT;
- interact with IBM Watson API and Elasticsearch;
- develop and streamline ETL processes using Informatica or IBM Datastage;
- conduct experiments with CI/CD processes on Microsoft Azure.

### Syntax of HTTP-Request

HTTP requests follow this syntax:

to write a request, make sure you use the following **2 mandatory** elements:

- name of request (will also be used for the script).
    Перед названием запроса необходимо поставить три хэштега
    <br></br><span style={{color: '#aaaaaa'}}>### name of request</span><br></br>
- HTTP method und URL:
    <br></br><span style={{color: '#008000'}}>GET, POST, PUT or DELETE</span>
    <span style={{color: '#cd1c1c'}}>URL</span>

**optionally**, you can include:

- HTTP header(s):
    <br></br><span style={{color: '#c178db'}}>header name:  header value ...</span>
- <span style={{color: '#26bbae'}}>body</span>, то есть текстовый контент, например, в формате JSON.<br></br>
    Inside the request, prepend the request body with a blank line.

В URL, headers and body можно включать переменные обрамлённые
    в фигурные скобки (enclosed in double curly braces):
    &#123;&#123;<span style={{color: '#f9a156'}}>variable</span>&#125;&#125;. See next topic [HTTP Variables](HTTP%20Variables).

В любом месте допустимо использование однострочных комментариев, которые  необходимо предварить двойным slash.
<span style={{color: '#cccccc'}}>//comments</span>

Перед несколькими запросами можно указать название группы запросов,
это позволит, при наличии большого числа запросов, удобно работать со структурой, в том числе, быстро находить запросы.
Название группы запросов необходимо предварить одним или двумя хэштегами.
<br></br><span>## Название группы запросов</span>

1. Click the run button to execute the HTTP request.

2. Перед названием запроса необходимо поставить три хэштега.

3. Click the `Structure` button to see the requests structure, в том числе названия групп запросов.

4. На следующей строке после названия запроса необходимо указать HTTP method и URL, который может включать в себя переменные.

5. На следующей строке можно опционально написать HTTP request headers. Values of header могут включать в себя переменные.

6. Далее, пропустив одну строку, можно опционально написать HTTP request body.

7. После выполнения запроса в тексте появится строка со статусом ответа и управляющими ссылками-кнопками.
    Нажмите на дату и время ответа, если необходимо увидеть техническую информацию ответа, например, статус и заголовки ответа.

8. Click the `to left` or `to right` to see в левой или в правой панели the response body, как правило в формате json.
